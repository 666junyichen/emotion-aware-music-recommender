# 项目说明

## 基本信息

项目名称：Zen Music Recommender

时间：项目原型阶段

项目阶段：情绪感知音乐推荐原型 / 概念验证

角色：数据处理、模型实验、推荐逻辑、Web 原型实现、交互流程整理

使用工具：Python, Flask, NeuroKit2, pandas, scikit-learn, JSON, HTML, JavaScript

## 项目内容

负责的内容：

- 构建心率 BPM 到情绪状态的推荐流程，将用户输入转换为 `Happy`、`Sad`、`Calm` 等情绪标签。
- 使用 ECG 数据处理流程提取 HRV 和 BPM 特征，包括 `RMSSD`、`SDNN`、`LF_HF` 和平均 `BPM`。
- 使用 Random Forest 分类模型进行情绪预测实验，并将预测结果导出为 JSON 映射文件。
- 设计结构化音乐库，按情绪标签维护曲目、艺术家、valence、energy、tempo、音频路径和封面路径。
- 实现 Flask Web 原型，包括 BPM 输入、情绪匹配、音乐推荐、播放页面、停止重置和反馈弹窗。
- 整理公开项目文档，移除私人身份标识和本地环境路径等不适合公开的信息。

数据来源：

- ECG 情绪识别流程基于 WESAD 数据集的生理信号处理思路。
- 音乐推荐库来自项目内维护的 `music.json` 结构化元数据。

分析方法：

- 使用 NeuroKit2 处理 ECG 信号并提取心率变异性特征。
- 使用 `StandardScaler` 标准化特征。
- 使用 `LabelEncoder` 编码情绪标签。
- 使用 Random Forest 分类器训练情绪识别模型。
- 使用 BPM 和情绪预测结果驱动音乐推荐。

最终结果：

- 完成一个可运行的 Flask 音乐推荐原型。
- 支持用户输入 BPM 后获得情绪解释和匹配音乐。
- 完成 15 首音乐、3 类情绪的结构化推荐库。
- 生成 50 条 BPM 到情绪的预测映射。
- Notebook 中记录的模型训练输出准确率为 83.33%。

可以写进经历描述的量化成果：

- Built an emotion-aware music recommendation prototype using Flask, ECG-derived HRV features, and a Random Forest classifier.
- Processed ECG signals into `RMSSD`, `SDNN`, `LF_HF`, and `BPM` features for emotion classification.
- Achieved 83.33% training-notebook test accuracy in the recorded Random Forest experiment.
- Structured a 15-track music library across 3 emotion categories, with 5 tracks per category.
- Exported 50 BPM-to-emotion mappings to support recommendation logic in the web prototype.
- Implemented an end-to-end BPM input, emotion mapping, recommendation, playback, and feedback flow.
