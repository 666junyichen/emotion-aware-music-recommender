# PulseScape 项目现状与下一步

## 现在这个项目是什么

PulseScape 是一个基于用户当前心率 BPM 的情绪感知音乐推荐 Web Demo。用户输入 BPM 后，系统会推断当前听歌状态，再根据歌曲的情绪、BPM、energy、valence、genre、反馈记录等信息推荐音乐。

这个项目已经从原来的简单 Flask demo 升级成更适合作品集展示的体验：

- 用户可以输入 BPM。
- 系统会把 BPM 映射到 Calm、Sad、Happy、Focused、Recovery 等状态。
- 系统会用可解释评分推荐歌曲，而不是随机选歌。
- 推荐页有沉浸式背景图和音乐播放器。
- 页面会解释为什么推荐这首歌。
- 页面会展示相似音乐。
- Library 和 Similar tracks 卡片可以点击进入播放器。
- 已经接入 10 首可公开播放的 MP3，放在 `static/audio/public/`。
- 已经用 Pixabay API 下载 25 张本地 scene images，放在 `static/images/scenes/`。
- 已经增加 Like、Not for me、听后 BPM 反馈。
- 已经增加可选 MongoDB Atlas 持久化，配置后会把反馈写入 `pulsescape.feedback`。
- 已经增加 Vercel 部署配置。

## 已完成内容

### 1. 后端推荐逻辑

`app.py` 现在包含：

- BPM 输入处理。
- BPM 到情绪状态的 fallback 映射。
- 从 `predictions.json` 读取已有预测结果。
- 歌曲评分推荐逻辑。
- Why this track 推荐解释。
- Similar tracks 相似歌曲列表。
- Library 页面路由。
- `/track/<track_id>` 播放路由。
- `/feedback` 反馈路由。

推荐评分大致考虑：

- 情绪是否匹配。
- 用户 BPM 和歌曲 BPM 是否接近。
- 歌曲 energy 是否适合当前状态。
- 歌曲 valence 是否适合当前状态。
- 用户当前会话内的 like/dislike 反馈。
- 少量随机权重，让同一 BPM 不会永远完全一样。

### 4. MongoDB 状态

项目现在可以复用已有的 free MongoDB Atlas cluster，不需要再创建付费 cluster。

推荐配置：

```text
MONGODB_URI=mongodb+srv://...
MONGODB_DB=pulsescape
```

本地 `.env` 已经加入 `.gitignore`，不会提交。`.env.example` 只保留变量模板。

当前已实现：

- 没有 `MONGODB_URI` 时，项目照常运行。
- 有 `MONGODB_URI` 时，反馈事件会写入 MongoDB。
- 数据库名使用 `pulsescape`。
- collection 使用 `feedback`。
- 已验证 Atlas `ping` 成功。

### 5. Pexels / Pixabay API 状态

项目有脚本：

```text
scripts/fetch_scene_images.py
```

Pexels 命令：

```powershell
$env:PEXELS_API_KEY="your-key"
python scripts/fetch_scene_images.py pexels
```

Pixabay 命令：

```powershell
$env:PIXABAY_API_KEY="your-key"
python scripts/fetch_scene_images.py pixabay
```

当前状态：

- Pexels key 测试时返回 `HTTP Error 403: Forbidden`。
- 这通常表示 key 未激活、复制错误、邮箱/条款未完成，或应用权限未生效。
- Pixabay key 已经测试成功。
- Pixabay 已经用于下载 25 张本地场景图并更新 `music.json`。

这两个 API 对项目都有帮助：

- Pexels 更适合情绪化、高质量照片搜索。
- Pixabay 可用，而且文档明确要求不要长期 hotlink，所以更适合下载到本地静态资源后使用。

## 还可以继续改进的内容

### 1. Pexels key 重新验证

需要在 Pexels 后台确认：

- 邮箱是否验证。
- API application 是否 active。
- Terms 是否接受。
- key 是否完整复制。

如果仍然 403，建议重新生成一个 key。

### 2. 收藏持久化

现在收藏仍在浏览器 `localStorage`。后续可以把收藏写入 MongoDB：

```text
pulsescape.favorites
```

### 3. 推荐事件记录

可以新增 collection：

```text
pulsescape.recommendation_events
```

记录用户输入 BPM、预测情绪、推荐歌曲、score breakdown，用于项目报告和后续算法分析。

### 4. 音频特征自动分析

现在 BPM、energy、valence 主要来自 metadata。未来可以加入 Librosa 或其他音频分析工具，自动提取节奏和音频特征。
