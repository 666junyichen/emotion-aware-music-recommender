# PulseScape 项目现状与下一步

## 现在这个项目是什么

PulseScape 是一个基于心率 BPM 的情绪感知音乐推荐 Web Demo。用户输入当前心跳频率后，系统会推断一个情绪/状态标签，再根据歌曲的情绪、BPM、energy、valence 等元数据推荐音乐。

现在项目已经从原来的简单 Flask demo 升级成更像产品的体验：

- 用户可以输入 BPM。
- 系统会把 BPM 映射到 Calm、Sad、Happy、Focused、Recovery 等状态。
- 系统会基于评分逻辑推荐歌曲，不再只是随机选一首。
- 推荐页面有沉浸式背景图/情景图。
- 每首歌有 tempo、energy、valence、genre、vocal type、scene image 等字段。
- 页面会解释为什么推荐这首歌。
- 页面会展示类似音乐。
- Library 和 Similar tracks 的卡片现在可以点击进入播放器，用户可以直接播放对应歌曲。
- 用户可以在浏览器本地收藏歌曲。
- 有 Library 页面展示音乐库。
- 已增加 Vercel 配置，方便部署成可访问网站。

`app.py` 现在包含：

推荐评分大致考虑：

- 情绪是否匹配。
- 用户 BPM 和歌曲 BPM 是否接近。
- 歌曲 energy 是否适合当前状态。
- 歌曲 valence 是否适合当前状态。
- 少量随机权重，让同一 BPM 不会永远完全一样。

## 关于背景图/情景图怎么做

你不一定要手动上传每张背景图。有三种方法：

### 方案 A：先用稳定远程占位图

当前项目先使用类似下面这种 URL：

```text
https://picsum.photos/seed/song-id/1600/1000
```

优点：

- 不需要 API key。
- 每首歌可以稳定得到不同图片。
- 很适合 demo 阶段。

缺点：

- 图片不一定和歌曲语义完全对应。
- 授权和 attribution 不如正式图库 API 清晰。

### 方案 B：使用 Pixabay / Pexels API

这是更适合正式发布的方案。

做法：

1. 每首歌写一个 `scene_query`，比如 `rainy window piano`。
2. 后端或构建脚本调用 Pixabay/Pexels API 搜图。
3. 把返回的图片 URL、作者、来源页、license 写回 `music.json`。
4. 前端只读取 `scene_image`。

优点：

- 图片更贴合歌曲情绪。
- 可以记录来源和授权。
- 更适合公开展示。

缺点：

- 需要注册 API key。
- 要遵守 API rate limit 和 attribution 规则。

### 方案 C：手动精选少量高质量图

如果你想让作品集最好看，可以每种 mood 先手动选 3-5 张高质量图片。

优点：

- 视觉效果最好。
- 最容易保证每首歌的情景一致。

缺点：

- 前期需要人工挑选。
- 每张图都要确认授权。

我建议：现在用方案 A 快速部署；之后用方案 B 或 C 替换成真正匹配音乐的图片。

## 还没有完成的内容

### 1. 真正可公开发布的音乐素材

现在项目保留了原来的音乐 metadata，但原始音频是否可以公开发布还需要 license review。当前浏览器播放的是 demo audio fallback。

已经新增一个候选清单：

```text
data/public_music_candidates.json
```

里面整理了第一批适合这个项目的 Pixabay Music 候选歌曲，包括标题、作者、情绪标签、genre、vocal type、license、source_url 和 attribution。

下一步需要：

- 优先从 `data/public_music_candidates.json` 里的 Pixabay source_url 打开歌曲页面。
- 手动下载 MP3。
- 放到 `static/audio/public/`。
- 记录来源 URL、license、artist、attribution。
- 在 `music.json` 里替换 `filepath`，例如 `audio/public/please-calm-my-mind.mp3`。
- 确认是否允许公开部署。

### 2. 更精准的背景图

现在每首歌已经有 `scene_image` 字段，但还可以继续升级：

- 加 `scene_query`。
- 用 API 自动搜索图片。
- 保存真实图片来源。
- 给每首歌匹配更准确的视觉情景。

已经新增脚本：

```text
scripts/fetch_scene_images.py
```

用 Pexels：

```powershell
$env:PEXELS_API_KEY="your-key"
python scripts/fetch_scene_images.py pexels
```

用 Pixabay：

```powershell
$env:PIXABAY_API_KEY="your-key"
python scripts/fetch_scene_images.py pixabay
```

这个脚本会读取 `music.json` 里的 `scene_query`、`scene_theme` 或 `genre`，调用图库 API，写回 `scene_image`、`scene_source_url`、`scene_attribution` 和 `scene_license`。

### 3. 收藏推荐还只是本地体验

现在收藏存在浏览器 `localStorage`，刷新不会丢，但换设备会丢。

如果要做成更完整产品，可以加：

- 用户登录。
- 数据库。
- 收藏表。
- 反馈表。
- 基于收藏历史的推荐权重。

### 4. 音乐库还可以扩展类型

现在主要还是轻音乐/纯音乐方向。可以加入：

- Gentle vocal
- Lofi
- Jazz
- Acoustic
- Soft pop
- Cinematic
- Electronic chill
- Nature soundscape

每首歌都需要补齐 BPM、energy、valence、genre、vocal type。

### 5. 算法还可以更像推荐系统

目前是透明评分逻辑，适合作品集和 demo。之后可以继续做：

- 用户反馈影响推荐。
- 喜欢/不喜欢影响相似歌曲。
- 音乐特征自动提取。
- 根据听完后的二次 BPM 变化评估是否真的有调节效果。
